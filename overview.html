Module 0

Core Philosophy

Designed for the common case: fast on the likely path, bounded on the hard path, and explainable all the way through.

Evidence First

OCR, vision scores, recent frames, and search snippets are gathered as evidence before the system commits to a narrow taxonomy leaf.

Bounded Decisions

Each stage asks a smaller question: broad classification, family selection, leaf rerank, or search-backed correction. That keeps later repairs controlled.

Observable

Stages, attempts, artifacts, and reasoning are persisted so operators can inspect what ran, what was rejected, and what actually became final.

Module 4

Prompting and LLM Contracts

Scenalyze does not use one giant vague prompt. It uses task-specific prompts and task-specific JSON schemas so each model call answers one bounded question.

Base Classification Prompt

The initial classifier combines noisy OCR text with recent evidence frames. The prompt teaches the model to treat OCR as hints, not ground truth, and to avoid over-weighting isolated logo-only end cards.

OCR stance: extracted text is machine-generated evidence and can be incomplete, corrupted, or duplicated.
Image stance: recent frames are attached in chronological order so earlier product shots can outweigh later carrier or partner logos.
Output contract: the model must return strict JSON with brand, category, confidence, and reasoning.

Why Separate Contracts Matter

Once the pipeline starts reranking taxonomy leaves or selecting a family, freeform outputs become dangerous. Scenalyze narrows the schema along with the question.

Initial Classification
brand, category, confidence, reasoning
Family Selection
family_index, confidence, reasoning
Category Rerank

Entity Grounding
entity_name, entity_kind, genres, confidence, reasoning
Module 5

Taxonomy Selection and Reranking

The raw category from the first LLM pass is not yet the final taxonomy answer. Scenalyze narrows broad guesses into the canonical tree using retrieval, branch selection, and leaf reranking.

Step 1

Raw Category

The initial model may say something broad like Banking, Movie, or Food & Beverage.

Step 2

Embedding Retrieval

The mapper pulls nearby canonical candidates from the taxonomy rather than trusting the broad phrase directly.

Step 3

Broadness Check

If neighbors scatter across unrelated families, the system treats the input as structurally broad instead of pretending it already has a leaf.

Step 4

Family Selection

The model sees a bounded numbered menu of families and returns family_index. This picks the branch before the leaf.

Step 5

Leaf Rerank

Inside the chosen family, the model reranks a constrained candidate set and returns category_index instead of inventing a new label.

Why Branch Before Leaf?

This is the main defense against specificity jumps. Without a branch-first stage, a broad phrase like Movie can fall into a wrong leaf such as Movie Theatres, and a broad phrase like Food & Beverage can jump straight to Alcoholic beverages.

The key teaching idea is simple: first prove the right branch, then prove the right leaf within that branch.

Module 6

Web Search, Brand Review, and Grounding

Search is evidence enrichment, not direct taxonomy selection. It is used when the system needs help confirming a brand, grounding a media title, or narrowing an overly broad category.

Brand Review

If the predicted brand looks weakly anchored, the pipeline can trigger a bounded web confirmation pass. The result keeps ambiguity metadata so operators can see what was resolved and why.

Entity Grounding

Media-like ads can trigger title-oriented grounding so the system figures out what the advertised thing is before choosing the taxonomy branch.

Specificity Rescue

If a result stays too broad after mapping, search snippets can add discriminative evidence that helps the system choose a narrower but still supported taxonomy label.

Important Boundary

Search snippets are stored with source-aware structure like title, body, and URL. They are treated as evidence alongside OCR and frames. They do not get to bypass the bounded selection steps and directly invent a final category.

Module 7

The Rescue Ladder

When the fast path fails, Scenalyze escalates through bounded fallback stages instead of jumping straight to an unrestricted retry.

1

Initial Tail Pass

Recent tail frames, OCR evidence, and multimodal classification. This is the normal path, not a rescue.

fast path
2

OCR Rescue

Expands the evidence window and retries with a denser OCR profile when the initial pass lacks usable text anchors.

bounded retry
3

Express Rescue (Vision)

A vision-heavier fallback when text evidence is weak but visual cues may still support the brand or category story.

vision rescue
4

Family Selection and Leaf Rerank

Used when the broad answer is plausible but the final taxonomy leaf still needs a constrained branch-first correction.

taxonomy repair
5

Search-Backed Specificity Rescue

Adds external evidence only when the current category is too broad or the brand/entity interpretation still looks under-grounded.

evidence enrichment
6

Extended Tail or Full Video

The expensive option. The system widens temporal coverage only after the cheaper and more targeted corrections fail.

last resort
